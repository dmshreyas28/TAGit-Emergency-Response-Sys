# TAGit Mobile App – Tech Stack

This document outlines the technologies, libraries, and tooling used by the TAGit mobile app.

## Overview

- Platform: Flutter (Material 3)
- Targets: Android, iOS (desktop/web folders are present but mobile is primary)
- Architecture: Provider (ChangeNotifier) + Firebase backend
- Core features: NFC emergency data, authentication, profile/docs management, offline-friendly UI

## Languages & SDKs

- Dart SDK: ^3.5.0
- Flutter: Stable 3.x (Material 3 enabled)

## State Management & Architecture

- Provider (^6.1.1) for app-wide state and services
  - Services include: `AuthService`, `FirestoreService`, `NFCService`, `LocationService`, `ConnectivityService`
- Auth wrapper pattern (Consumer<AuthService>) for navigation

## UI/UX

- Material 3 components and themes (see `lib/utils/app_theme.dart`)
- Responsive layout: `flutter_screenutil` (^5.9.0)
- Iconography & SVG rendering: `flutter_svg` (^2.0.0)
- Google Fonts (Poppins-ready): `google_fonts` (^6.3.2)
- Bottom navigation: `salomon_bottom_bar` (^3.3.2)

## Firebase & Backend

- `firebase_core` (^3.6.0)
- `firebase_auth` (^5.3.1)
- `cloud_firestore` (^5.4.4)
- `firebase_storage` (^12.3.4)
- Firestore/Storage rules managed in the website repo as reference; mobile app uses client SDKs.

## Device & Platform APIs

- NFC: `nfc_manager` (^4.1.1), `ndef` (^0.3.4), `nfc_manager_ndef` (^1.1.0)
- Geolocation: `geolocator` (^11.0.0)
- Permissions: `permission_handler` (^11.2.0)
- Connectivity: `connectivity_plus` (^6.0.5)
- URL intents: `url_launcher` (^6.3.2)

## Media, Storage & Data

- Image/file picking: `image_picker` (^1.0.7), `file_picker` (^8.0.0+1)
- Local preferences: `shared_preferences` (^2.2.2)
- Local database: `sqflite` (^2.3.2)
- QR generation (where used): `qr_flutter` (^4.1.0)
- Utility libs: `intl` (^0.18.1), `uuid` (^4.3.3)

## Navigation

- Auth-driven routing via `AuthWrapper` in `lib/main.dart`
- Main app shell: `MainScreen` with Salomon bottom bar

## Theming

- Centralized theme in `lib/utils/app_theme.dart`
  - Primary brand: Coral `#FF7643` (aliases preserved as `primaryRed` for legacy code)
  - Button/Input/Card themes with consistent radii and spacing

## Build & Tooling

- Flutter build/run via `flutter run`, `flutter build <platform>`
- Testing: `flutter_test`
- Lints: `flutter_lints` (^5.0.0)

## Project Layout (high level)

- `lib/`
  - `main.dart` (Auth wrapper + routes)
  - `screens/` (auth, main, nfc, profile, documents, settings, etc.)
  - `services/` (auth, firestore, nfc, location, connectivity, storage)
  - `utils/` (theme, snackbar helper, offline indicator)
  - `widgets/` (reusable components)
- `android/`, `ios/` (native shells)

## Notes

- Desktop/web folders are present (generated by Flutter) but the primary deployment is mobile.
- Social login UI placeholders exist; core auth is email/password.

See the website stack in `../tagit_website/TECH_STACK.md` for the Next.js app.
